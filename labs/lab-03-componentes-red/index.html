    Laboratorio 03 - Componentes de Red

    Contexto

    Identificar dispositivos finales (hosts) e intermedios (switch, router), interfaces de red y MTU.

    Objetivo

    • Listar interfaces de red en el sistema.
    • Ver tabla ARP.
    • Simular un switch con bridge de Linux.

    Topologia

    graph TD
    PC1[PC1] --> SW[Switch]
    PC2[PC2] --> SW
    SW --> R1[Router]
    R1 --> Internet

    Implementacion en Linux

    1. Mostrar interfaces: 
      ip link show
    2. Mostrar tabla ARP: 
      ip neigh show
    3. Crear un bridge (switch virtual) y conectar dos namespaces: 
      sudo ip link add br0 type bridge
sudo ip link set br0 up
sudo ip netns add ns1
sudo ip netns add ns2
sudo ip link add veth1 type veth peer name veth1-br
sudo ip link add veth2 type veth peer name veth2-br
sudo ip link set veth1 netns ns1
sudo ip link set veth2 netns ns2
sudo ip link set veth1-br master br0
sudo ip link set veth2-br master br0
sudo ip link set veth1-br up
sudo ip link set veth2-br up
sudo ip netns exec ns1 ip addr add 10.0.0.1/24 dev veth1
sudo ip netns exec ns2 ip addr add 10.0.0.2/24 dev veth2
sudo ip netns exec ns1 ip link set veth1 up
sudo ip netns exec ns2 ip link set veth2 up
    4. Verificar conectividad entre namespaces: 
      sudo ip netns exec ns1 ping 10.0.0.2

    Validacion

    • El ping entre namespaces debe funcionar.
    • bridge fdb show muestra las MACs aprendidas.

    Notas profesionales

    • En Windows: getmac, arp -a.
    • En Cisco: show mac address-table.
